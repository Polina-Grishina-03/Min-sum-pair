# Поиск пары целых чисел с минимальной суммой
## Условие:
Найти два целых числа в заданном массиве, сумма которых минимальна.

## Решение:
### Код на Python:

```python
def min_sum_pair(arr):
    # 1. Для начала оставим только целые числа в заданном массиве
    integers = [x for x in arr if isinstance(x, int)]
    
    # 2. Проверим, что у нас есть минимум 2 целых числа для нахождения их суммы
    if len(integers) < 2:
        raise ValueError("В массиве должно быть не менее двух целых чисел")
    
    # 3. Отсортируем элементы по возрастанию
    integers.sort()
    
    # 4. Теперь первые 2 элемента - это наша пара с минимальной суммой
    min_pair = (integers[0], integers[1])
    min_sum = integers[0] + integers[1]
    
    return min_pair, min_sum
```

### Как работает алгоритм:
**1. Фильтрация**\
Проходим по всем элементам входного массива.
Оставляем только те, чей тип - int (отсеиваем str, float, None и др.).\
**2. Проверка валидности**\
Если осталось меньше 2 целых чисел - выбрасываем ошибку.\
**3. Сортировка**\
Упорядочиваем целые числа по возрастанию.
После сортировки два наименьших числа окажутся на первых позициях.\
**4. Формирование результата**\
Берем первые два элемента отсортированного массива - это искомая пара.
Считаем их сумму.
Возвращаем пару и сумму.

### Анализ временной сложности O(n):
1. Фильтрация целых чисел проходит по всем n элементам массива - имеем O(n).
2. Проверка длины оставшегося массива - O(1).
3. Сортировка методом <list>.sort() - это стандартная сортировка в Python с помощью гибридного алгоритма сортировки Timsort - время работы O(n log n).
4. Выбор пары и суммы (доступ к первым двум элементам) - O(1).

Итоговая временная сложность: O(n) + O(1) + O(n log n) + O(1) ≈ O(n log n) - рассматриваем худший исход, когда почти все элементы целые.

**Вывод**: асимптотическая сложность - **O(n log n)** из-за сортировки.
